# 答题中题型与题序导航设计

## 目标

刷题和自测过程中，用户不再只能依赖“上一题 / 下一题”，而是可以随时按题型查看题号并跳到任意一道题。两种模式同时支持顺序与乱序。

## 交互

- 侧栏提供“题型与题序”面板，题号按钮代表当前答题序列中的位置。
- “全部 / 单选 / 多选 / 判断 / 填空 / 简答 / 论述”仅筛选题号面板，不删除题目或作答。
- 点击题号立即跳转；当前题与已答题使用不同状态色。
- 顺序与乱序切换时保留当前题，避免用户失去答题上下文。
- 自测把 `questionOrder`、`orderMode` 和 `currentIndex` 写入现有会话设置，刷新或重新进入时恢复同一序列。

## 结构

- `questionNavigation.ts`：生成、恢复和映射题目顺序的纯函数。
- `QuestionNavigator.tsx`：刷题与自测共享的导航面板。
- 刷题顺序只在当前页面会话中保留；自测顺序随持久化会话保存。

## 验证

- 领域测试覆盖顺序生成、无重复乱序、旧会话恢复和题型筛选。
- 浏览器验收覆盖题号跳转、题型筛选、乱序切换及自测重新进入后的恢复。
