# Quiz Studio 首版缺口补齐设计

## 范围

本轮补齐技术设计书中“首版必须完成”的功能，但不制作正式安装包、签名或自动更新。保留本地优先、前端不直连 SQLite、AI Provider 可替换三项既有约束。

## 实施策略

采用纵向闭环分批交付。每一批同时完成领域模型、Rust/SQLite、浏览器开发仓库、React 界面和自动化测试，确保用户场景可独立验收。

1. 数据资产与题库编辑：搜索、完整编辑、题库导出和恢复。
2. 自测闭环：会话、统一提交、本地评分、答题记录、恢复、错题与结果分析。
3. 文档导入：DOCX、文本 PDF、图片/扫描 PDF，经统一 `DocumentBlock -> QuestionDraft` 管线进入校正工作台。
4. AI 主观题评分：评分点、结构化输出、自动计分、错误恢复和费用提示。
5. OCR/AI 任务恢复、附件存储、跨平台构建检查和可访问性收尾。

## 数据流与边界

题库导出使用带格式版本号的 JSON 容器，扩展名为 `.qbank`。恢复时先做 Zod/Serde 校验，再通过单一事务写入；默认生成新 ID，避免覆盖已有数据。搜索首先采用本地字段匹配，桌面端后续接入已有 FTS5 表。完整题目编辑复用导入校正编辑器的领域规则，但不复用其页面状态。

自测以 `test_sessions` 和 `attempts` 为唯一持久化来源。客观题本地评分；主观题允许显示答案并记录状态，AI 评分作为可选增强，不阻塞提交。DOCX/PDF 解析器只负责产生统一文档块，不直接写题库。

## 错误处理与验证

所有导入、恢复、批量写入都必须原子化；格式、版本、空题库、重复 ID 和外键错误向用户显示可操作信息。新增领域行为遵循测试先行，最终执行前端测试、生产构建、Rust 格式检查、Rust 编译和后端测试。
