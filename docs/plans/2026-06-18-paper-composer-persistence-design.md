# 自定义组卷与答题工作区持久化设计

## 用户流程

首次进入某题库的刷题或自测时显示组卷器。用户可以限定原题号范围、为每种题型填写数量，再在候选清单中精确勾选题目并上下调整顺序。开始后进入现有答题页；已有未清理工作区时直接恢复，不重复组卷。

答题侧栏新增“清空答案”和“换卷”。普通跳题、切换应用页面、关闭并重新打开应用都保留当前试卷、题序、当前位置、答案、已确认状态和参考答案显示状态。“清空答案”仅清除作答状态，保留试卷；“换卷”在用户确认后清除当前工作区并返回组卷器。

## 数据结构

组卷规则由纯函数处理，输入完整题库、题号范围、各题型数量和抽取方式，输出无重复题目 ID。手工调整后的 ID 顺序直接作为试卷定义保存。

每个 `mode + bankId` 对应一个本地工作区，内容包含 `questionOrder`、`orderMode`、`currentIndex`、`responses`、`submitted`、`revealed`。浏览器与 Tauri WebView 都使用同步 localStorage 保存工作区，保证快速切页不会丢最后一次输入；自测同时继续写入 SQLite 会话，保留既有评分与历史能力。

## 边界处理

- 题库题目删除后，恢复时丢弃失效 ID；新增题目不会偷偷加入已组好的卷子。
- 题型数量超过范围内可用数量时取全部可用题，并在组卷器显示实际选题数。
- 组卷结果为空时禁止开始。
- 换卷和清空答案均二次确认，避免误操作。
