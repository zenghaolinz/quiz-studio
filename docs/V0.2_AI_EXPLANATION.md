# v0.2.0：原文预览修复与 AI 解析补全

## 原文预览根因

旧版把 `DocumentBlock.kind` 直接拼进 CSS class：

```tsx
className={`source-line ${block.kind}`}
```

当 block 是选项时会得到 `class="source-line option"`，与刷题页全局 `.option` 样式发生冲突。
`.option > span` 会强制所有直接子 `span` 宽高为 28px，因此原文文本列被压成约 28px，中文只能逐字换行，看起来像重叠。

v0.2.0 改为 `source-line--option` 等命名空间类，并把左栏重构为固定元信息列 + 弹性文本列。

## AI 解析功能

题库详情页新增“AI 补全题目解析”：

- 统计已有解析与缺失解析数量；
- 只处理 `explanationMarkdown` 为空的题目，不覆盖已有解析；
- 支持简洁、详细、分步推导三种风格；
- 支持 1～4 并发；
- 支持完成当前请求后暂停；
- 支持单题生成；
- 每题成功后立即写回 SQLite；
- 失败不会清除已成功结果；
- 题库列表中可展开查看解析。

设置页新增语言模型 Provider 配置：

- OpenAI
- Anthropic Claude（原生 Messages API）
- Google Gemini（OpenAI 兼容入口）
- DeepSeek
- 智谱 GLM
- 阿里云百炼 / Qwen
- Moonshot / Kimi
- 火山方舟
- Ollama
- LM Studio
- 自定义 OpenAI Compatible

Base URL 仅作为预设，用户可以修改。模型名不写死，避免厂商模型更新后配置失效。
API Key 继续由系统凭据库存储，不写入 SQLite。

## 本地验证

```powershell
npm install
npx tsc -b
npx vitest run
npm run build

cd src-tauri
cargo fmt --check
cargo check
cargo test
cd ..

npm run tauri:dev
```

手工验证：

1. 导入含多道选项题的 Markdown，确认左侧文字不再逐字堆叠。
2. 在设置页选择厂商预设，填写模型与 API Key，保存并测试连接。
3. 打开题库，展开“AI 补全题目解析”。
4. 先对一道题单独生成，再批量补全剩余题目。
5. 重启应用，确认解析仍存在。
